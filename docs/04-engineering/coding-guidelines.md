# Standards de Code

## Principes

- Fonctions courtes, responsabilite unique.
- Types explicites sur fonctions publiques.
- Nommage clair des symboles.
- Eviter duplication et effets de bord.

## Regles projet

- Parsing intent separe de la couche interface.
- Ne pas casser un intent existant sans mise a jour des scenarios.
- Ajouter tests pour toute logique metier nouvelle.

## Qualite

- Lancer tests avant validation.
- Mettre a jour documentation impactee.
- Ne pas introduire de secrets en dur.
