# Tests et documentation

## Tests

- Ecrire des tests unitaires pour la logique metier importante.
- Ajouter des tests d'integration pour les flux critiques.
- Tester les cas limites et les erreurs, pas uniquement le chemin nominal.

## Documentation

- Documenter les decisions importantes (architecture, conventions).
- Ajouter des docstrings sur les fonctions non triviales.
- Tenir le README a jour avec installation, usage et commandes utiles.

## Revue de code

- Faire relire chaque changement significatif.
- Prioriser en revue: bugs, regressions, securite, maintenabilite.
- Donner des retours precis, respectueux et orientes solution.
